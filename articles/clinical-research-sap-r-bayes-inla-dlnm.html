臨床研究SAP作成：R, ベイズ統計, INLAとDLNM活用

Photo from Unsplash

皆さん、こんにちは！「臨床統計ラボ」の管理人です。10年以上にわたり、製薬企業やCROで数多くの臨床試験の統計解析に携わってきました。

臨床研究の信頼性と透明性を確保する上で、統計解析計画書（SAP: Statistical Analysis Plan）の作成は非常に重要です。そして近年、Rを用いた高度な解析手法、特にベイズ統計、INLA（Integrated Nested Laplace Approximation）、そして分布ラグ非線形モデル（DLNM: Distributed Lag Non-linear Model）が生物統計の分野、特に臨床研究で注目されています。この記事では、ICH E9ガイドラインに準拠したSAPの作成方法から、これらの最新かつ強力な統計手法をSAPにどのように組み込むかまで、実践的に解説していきます。

この記事でわかること

  • ICH E9ガイドラインに準拠した統計解析計画書（SAP）の基本構成と記載のポイント
  • ベイズ統計の基礎と、計算効率の高いINLAを用いた解析計画の立て方
  • 曝露の遅延効果や非線形性を評価する分布ラグ非線形モデル（DLNM）の臨床研究への適用方法
  • RとSASを用いた具体的な統計解析手法のコード例とSAPでの記述方法
  • SAP作成における実務的なヒントと海外の最新動向

結論から言うと

統計解析計画書（SAP）は、臨床試験の結果の信頼性と透明性を担保するために不可欠な文書です。ICH E9（統計的原則）に則り、主要評価項目や副次評価項目に対する解析手法、感度分析などを詳細かつ明確に記述することが求められます。近年では、Rを活用したベイズ統計、特にINLAや、曝露の遅延効果を捉えるDLNMといった先進的な解析手法を計画段階から適切に検討しSAPに組み込むことで、より深い臨床的洞察と効率的な解析が可能になります。

統計解析計画書（SAP）とは？

統計解析計画書（SAP: Statistical Analysis Plan）とは、臨床試験のプロトコルで規定された統計解析の方針を、より詳細に記述した文書のことです。臨床試験の実施前に作成され、試験デザイン、データ収集方法、主要評価項目（Primary Endpoint）、副次評価項目（Secondary Endpoint）などに対する具体的な統計解析手法を明確にします。これにより、解析における恣意性（しいてきせい）を排除し、解析結果の客観性と信頼性を保証します。

ICH E9ガイドライン「臨床試験のための統計的原則」（Statistical Principles for Clinical Trials）は、SAP作成の国際的な標準となっています。このガイドラインでは、解析対象集団（Analysis Set）、欠測データ（Missing Data）の取り扱い、多重性（Multiplicity）の調整方法、サブグループ解析（Subgroup Analysis）など、SAPに記載すべき項目が具体的に示されています。

SAPはプロトコルと密接に関連していますが、プロトコルが試験全体の計画を示すのに対し、SAPは特に「統計解析」の部分に特化して詳細を詰めます。プロトコルで大枠が決定された後、統計家が中心となってSAPを作成し、最終的には治験依頼者や規制当局の合意を得て確定されます。

ベイズ統計、INLA、分布ラグ非線形モデル（DLNM）の統計的背景・理論

ベイズ統計の基礎とINLA

従来の頻度論的統計学（Frequentist Statistics）が、観察されたデータの下で仮説の確率を評価するのに対し、ベイズ統計（Bayesian Statistics）は、事前情報（事前分布: Prior Distribution）とデータから得られる情報（尤度: Likelihood）を統合して、より新しい情報（事後分布: Posterior Distribution）を導き出す統計的枠組みです。ベイズの定理は次のように表されます。

$$P(\theta | D) = \frac{P(D | \theta) P(\theta)}{P(D)}$$

ここで、$P(\theta | D)$ はデータ $D$ が与えられたときのパラメータ $\theta$ の事後分布、$P(D | \theta)$ は尤度、$P(\theta)$ は事前分布、$P(D)$ はデータの周辺尤度（規格化定数）です。

臨床研究において、過去の知見や専門家の意見を事前分布として組み込める点は大きなメリットです。特に、希少疾患や小規模試験、リアルワールドデータ（RWD）解析など、データが限られている状況で、よりロバストな推論を可能にします。しかし、複雑なモデルの事後分布を計算するには、MCMC（Markov Chain Monte Carlo）法などのサンプリング手法が用いられ、計算に時間がかかることが課題でした。

そこで注目されているのが、INLA（Integrated Nested Laplace Approximation）です。INLAは、MCMCのようなサンプリングに頼らず、ラプラス近似（Laplace Approximation）を用いて事後分布を直接的に近似する手法です。これにより、MCMCに比べて格段に高速で、大規模なデータセットや複雑な階層モデル（Hierarchical Models）に対しても効率的にベイズ推論を行うことができます。SAPに「ベイズモデルを使用するが、計算効率と実用性を考慮し、INLAを用いて推定を行う」と記載することで、解析の妥当性を高めることが可能です。

分布ラグ非線形モデル（DLNM）

多くの臨床現象では、曝露（Exposure）の影響が即座に現れるとは限りません。例えば、ある薬剤を服用してから副作用が現れるまでに時間差があったり、PM2.5のような環境因子が健康に与える影響が、曝露後数日〜数週間にわたって遅れて現れたりすることがあります。また、その影響は線形ではなく、曝露量に応じて非線形に変化することも少なくありません。

分布ラグ非線形モデル（DLNM: Distributed Lag Non-linear Model）は、このような「曝露の遅延効果（Lag Effect）」と「曝露-反応関係の非線形性（Non-linear Relationship）」を同時にモデリングするための強力な統計手法です。DLNMは、曝露の時系列における各時点での効果を結合し、特定の曝露レベルに対する累積的な反応を評価することを可能にします。

DLNMのコアとなるのは、曝露量とラグ期間の2つの次元で定義される「クロスベーシス関数（Cross-basis Function）」です。これは、曝露の遅延効果と非線形効果を柔軟に表現できる基底関数（Basis Functions）の組み合わせであり、一般的に次のように表現されます。

$$Y_t \sim \text{Poisson}(\mu_t)$$ $$\log(\mu_t) = \alpha + s(\text{Exposure}t, \text{lag}_0) + s(\text{Exposure}{t-1}, \text{lag}1) + \dots + s(\text{Exposure}{t-L}, \text{lag}_L) + \text{covariates}_t$$

ここで、$Y_t$ は時点 $t$ におけるアウトカム、$s(\cdot, \cdot)$ は曝露とラグ期間に対する非線形関数、$L$ は最大ラグ期間です。このモデルは、時間依存性のあるアウトカム、特に感染症の発生率や死亡率、副作用の遅延発現など、様々な臨床研究に応用可能です。SAPにおいては、「薬剤曝露の遅延効果および非線形性を評価するため、分布ラグ非線形モデル（DLNM）を適用する」といった形で、解析手法の選定根拠とともに記述します。

SAS/Rコード実装

ここでは、SAPで記述する解析手法の例として、Rを用いたDLNMとINLAの組み合わせ、およびSASを用いた一般的な線形混合モデルのコード例を示します。実際のSAPには、具体的なコードを記載する代わりに、どのような解析手法を用いるかを詳細に記述します。

Rコード：分布ラグ非線形モデル（DLNM）とINLAの組み合わせ

Rのdlnmパッケージでクロスベーシス関数を生成し、INLAパッケージでベイズ推定を行います。ここでは、架空の時系列データ（例：日ごとの薬剤曝露量と副作用報告数）を用いて、曝露の遅延効果と非線形性を評価する例を示します。

# 必要なパッケージのインストールと読み込み
# install.packages(c("dlnm", "INLA", "tidyverse", "splines"))
library(dlnm)
library(INLA)
library(tidyverse)
library(splines)

# --- 1. サンプルデータの生成 ---
set.seed(123)
n_days <- 365 # 1年間のデータ

# 時系列データ
df_data <- tibble(
  date = seq(as.Date("2023-01-01"), by = "day", length.out = n_days),
  # 架空の薬剤曝露量（例：日ごとの投与量）
  exposure = pmax(0, rnorm(n_days, mean = 10, sd = 3)),
  # 気温など、共変量として扱う可能性のある因子
  temperature = rnorm(n_days, mean = 20, sd = 5)
) %>%
  # 遅延効果をシミュレートするため、曝露量を少し遅らせて効果があるように設定
  mutate(true_effect_lag1 = lag(exposure, 1) * 0.5, # 1日遅れの線形効果
         true_effect_lag3 = lag(exposure, 3)^1.2 * 0.01) %>% # 3日遅れの非線形効果
  replace_na(list(true_effect_lag1 = 0, true_effect_lag3 = 0)) %>%
  # アウトカム（例：副作用報告数） - ポアソン分布に従うと仮定
  mutate(log_mu = 1 + 0.1 * temperature + true_effect_lag1 + true_effect_lag3) %>%
  mutate(outcome = rpois(n_days, exp(log_mu)))

# データの一部表示
print(head(df_data))

# --- 2. クロスベーシス関数の定義 ---
# 曝露量の基底関数: cubic B-spline（自然スプライン）で非線形性を表現
# knots: 曝露量の特定のパーセンタイルにノット（結び目）を配置
# df_exposure: 自由度。ノットの数+1.
exposure_basis <- crossbasis(
  df_data$exposure,
  argvar = list(fun = "ns", df = 4), # Natural Splineで自由度4
  arglag = list(fun = "ns", df = 3, knots = c(1, 3, 7)) # ラグもNatural Splineで自由度3, 1, 3, 7日でノット
)

# クロスベーシス関数の次元を確認
print(dim(exposure_basis))

# --- 3. INLAモデルの構築と推定 ---
# INLAで使用するデータフレームを準備
inla_data <- df_data %>%
  mutate(day_index = 1:n())

# INLAのモデル式
# outcome ~ exposure_basis（DLNMの項） + temperature（共変量）
# + f(day_index, model = "ar1")：時間による自己相関（AR(1)モデル）をINLAのランダム効果で表現
# family = "poisson"：アウトカムがカウントデータ（ポアソン分布）の場合

formula_inla <- outcome ~ exposure_basis + temperature + 
                f(day_index, model = "ar1", hyper = list(theta = list(prior = "loggamma", param = c(1, 0.01))))

# INLAによるモデル推定
# verbose = TRUEで詳細な出力
model_inla <- inla(formula_inla, 
                   family = "poisson", 
                   data = inla_data, 
                   control.compute = list(dic = TRUE, waic = TRUE, cpo = TRUE, config = TRUE), 
                   control.predictor = list(compute = TRUE), 
                   verbose = TRUE)

# モデルサマリーの表示
print(summary(model_inla))

# --- 4. 結果の可視化 (DLNMの効果曲線) ---
# クロスベーシス関数の予測値を生成
pred_exposure <- crossbasis(
  x = seq(min(df_data$exposure), max(df_data$exposure), length = 20),
  argvar = list(fun = "ns", df = 4),
  arglag = list(fun = "ns", df = 3, knots = c(1, 3, 7))
)

# 予測結果を取得
pred_fit <- predict(model_inla, 
                    exposure_basis = pred_exposure, 
                    ci = TRUE, 
                    type = "response", 
                    interval = "HPD") # HPD (Highest Posterior Density) 信頼区間

# 特定の曝露量に対する累積効果をプロット（例：exposure=15のときのラグ効果）
# plot(pred_fit, "overall") # 全体的な効果
# plot(pred_fit, "lag", var = 15) # 曝露量15におけるラグ効果
# plot(pred_fit, "var", lag = 3) # ラグ3における曝露量効果

# 簡易的な可視化例：特定の曝露量での累積効果
# この部分の描画は複雑になるため、ここでは省略し、
# SAPでは「最終的な効果曲線はプロットで示す」と記述するに留めます。

SAPでの記述例：

主要評価項目に対する解析手法 疼痛スコアの変化量（連続量）の解析には、ベイズ階層モデルを適用する。具体的には、薬剤曝露量と反応の遅延効果および非線形性を評価するため、分布ラグ非線形モデル（DLNM）を構築し、INLA（Integrated Nested Laplace Approximation）を用いて事後分布を推定する。曝露量の基底関数には自然スプライン（自由度X）、ラグ期間の基底関数には自然スプライン（自由度Y）を使用する。モデルには、患者ごとのランダム効果および時間トレンドを考慮する。欠測データは、多重代入法（Multiple Imputation）により処理する。

SASコード：一般的な線形混合モデル

SAPで記述する解析手法として、最も一般的なものの1つである線形混合モデル（Linear Mixed Model）のSASコード例です。反復測定デザインのデータ解析などに用いられます。

/* --- 1. サンプルデータの生成 --- */
DATA ADPC;
  CALL STREAMINIT(12345);
  DO USUBJID = 1 TO 50; /* 50人の被験者 */
    TRT = IFCN(RANUNI(0) < 0.5, 'Placebo', 'Drug A'); /* 治療群 */
    DO VISIT = 1 TO 5; /* 5回の測定時点 */
      AVISITN = VISIT;
      AVISIT = CATS('Visit ',VISIT);
      RSPT = RANNORM(0); /* ベースラインからの個体差 */

      /* 治療効果をシミュレート */
      IF TRT = 'Drug A' THEN DO;
        IF VISIT = 1 THEN CHG = RANNORM(0)*2; /* ベースラインの変動 */
        ELSE CHG = 5 + 0.5 * VISIT + RSPT + RANNORM(0)*2; /* 治療薬群の経時的な改善 */
      END; 
      ELSE DO;
        IF VISIT = 1 THEN CHG = RANNORM(0)*2;
        ELSE CHG = 0.2 * VISIT + RSPT + RANNORM(0)*2; /* プラセボ群のわずかな改善 */
      END;
      OUTPUT;
    END;
  END;
RUN;

/* データの一部表示 */
PROC PRINT DATA=ADPC(OBS=10);
  TITLE 'ADPCデータセットの例';
RUN;

/* --- 2. 線形混合モデルの解析 --- */
PROC MIXED DATA=ADPC METHOD=REML;
  CLASS USUBJID TRT AVISIT;
  MODEL CHG = TRT AVISIT TRT*AVISIT / DDFM=KENWARDROGER;
  RANDOM USUBJID / TYPE=VC;
  REPEATED AVISIT / SUBJECT=USUBJID TYPE=UN;
  LSMEANS TRT*AVISIT / DIFF ADJUST=TUKEY SLICE=AVISIT PDIFF=ALL CL;
  ESTIMATE 'Overall Drug A vs Placebo' TRT 1 -1 / CL;
  TITLE '反復測定データに対する線形混合モデル解析';
RUN;

SAPでの記述例：

副次評価項目に対する解析手法 ベースラインからの平均収縮期血圧の変化量（連続量）の解析には、線形混合モデル（Linear Mixed Model for Repeated Measures; MMRM）を適用する。モデルには、固定効果として治療群（Treatment）、来院（Visit）、および治療群と来院の交互作用項（Treatment × Visit）を含める。被験者内の相関構造には非構造型（Unstructured）を仮定し、被験者ごとのランダム切片（Random Intercept）を含める。自由度の計算にはKenward-Roger法を用いる。各来院時点での治療群間差は最小二乗平均（Least Squares Means; LSMEANS）を用いて推定し、多重比較の調整にはTukey法を適用する。

実務での活用場面

これらの先進的な統計手法は、SAP作成の各項目で以下のように具体的に活用できます。

  1. 主要評価項目・副次評価項目に対する解析手法：

    • DLNMの適用： 薬剤曝露後の副作用発現までの遅延期間が臨床的に重要である場合、例えば「特定の有害事象の累積発生率に対する薬剤曝露の分布ラグ非線形効果を評価する」とSAPに明記します。これにより、従来の即時効果のみを評価するモデルでは見逃されがちな、時間遅れを伴う複雑な薬物動態学的・薬力学的関係性を捉えることができます。
    • INLAによるベイズ推定： 特に早期臨床試験（例：Phase I/II）での用量設定研究や、希少疾患を対象とした試験で、既存の知見を事前分布として組み込み、より確実な推論を行いたい場合に有効です。「用量反応関係の推定にはベイズ階層モデル（INLAを用いた推定）を適用する」と記述し、計算効率のメリットも強調できます。

  2. 感度分析（Sensitivity Analysis）：

    • モデルの仮定が結果に与える影響を評価するために、DLNMの基底関数の選び方（自由度やノットの位置）やINLAの事前分布の選択について、異なる仮定を用いた感度分析をSAPに計画します。これは、ICH E9 R1（Estimand and Sensitivity Analysis）が重視する、解析の頑健性（Robustness）を示す上で非常に重要です。

  3. 探索的解析（Exploratory Analysis）：

    • 主要・副次評価項目以外の、より複雑な時間依存性データや、複数の曝露因子が関与する解析において、DLNMやベイズ統計（INLA）は強力なツールとなります。例えば、「特定の生体指標の経時的変化に影響を与える複数の治療関連因子の分布ラグ効果を探索的に評価する」といった記述が考えられます。

SAPにこれらの高度な解析手法を記述する際は、なぜその手法が必要なのか、どのような臨床的意義があるのかを明確にすることが肝要です。{{internal_link:SAP作成の基本とICH E9ガイドライン}}も参考にしてください。

海外の最新動向

欧米の臨床統計コミュニティ、例えばPSI（Statisticians in the Pharmaceutical Industry）、ASA（American Statistical Association）、PHUSE（Pharmaceutical Users Software Exchange）などでは、ベイズ統計やRWD（Real World Data）解析における先進的な統計手法の活用が活発に議論されています。

特に、ICH E9 R1の「Estimandと感度分析」の枠組みの導入により、解析計画の透明性と柔軟性がこれまで以上に求められています。この中で、ベイズ統計は事前情報を活用できる柔軟性から、特にレギュラトリーサイエンスの分野で注目度が高まっています。FDAやEMAなどの規制当局も、特定の状況下でのベイズ統計の使用をガイドラインで認めるようになってきています。

また、計算負荷の高いMCMC法の代替としてINLAが広く受け入れられつつあり、特に公衆衛生分野での環境疫学研究（PM2.5と健康影響など）ではDLNMとINLAを組み合わせた解析が標準的になりつつあります。この流れは、臨床研究においても、複雑な薬剤動態や疾患進展パターンをより正確にモデリングするために加速すると考えられます。

これらの動向は、最新の統計ソフトウェア（R, Stanなど）の発展によって加速されています。{{internal_link:RとSASの比較と使い分け}}に関する議論も盛んです。例えば、PHUSEやPSIの年次カンファレンスでは、Rやベイズ統計、機械学習を用いた臨床解析事例が多数発表されており、日本の臨床統計担当者もこれらの情報にアンテナを張ることが重要です。

よくある質問（FAQ）

Q1: SAPはいつ作成し、誰が承認するのですか？

A1: SAPは、プロトコルが確定した後、通常は最初の被験者組み入れ前までに作成されます。作成は統計家が中心となって行いますが、治験依頼者（スポンサー）や、必要に応じて規制当局の合意・承認を得て最終化されます。試験開始後にSAPを変更する場合は、プロトコル改訂と同様に適切な手続きが必要です。

Q2: プロトコルとSAPの主な違いは何ですか？

A2: プロトコルは、臨床試験の全体的な計画（目的、試験デザイン、被験者の選択基準、介入、評価項目など）を記載する文書です。一方、SAPはプロトコルで大枠が示された「統計解析」の部分を、より詳細かつ具体的に記述した文書です。SAPでは、解析対象集団の定義、主要評価項目・副次評価項目に対する具体的な統計手法、欠測データの取り扱い、多重性調整、サブグループ解析、統計ソフトウェアの使用バージョンなどが明記されます。

Q3: RとSAS、どちらで解析計画を記述すべきですか？

A3: どちらのソフトウェアで解析するかは、その企業の標準やプロジェクトの要件によります。SASは製薬業界で長く使われてきた実績があり、規制当局への提出資料の作成において依然として主流です。Rは、最新の統計手法の実装が早く、柔軟性が高いという利点があります。SAPでは、どちらのソフトウェアを用いるか、およびそのバージョンを明記します。例えば、「主要解析はSAS（バージョンX.Y）を用いて実施し、探索的解析や感度分析の一部にはR（バージョンZ.W）を用いる」といった記述が考えられます。

Q4: ベイズ統計やDLNMをSAPに含める際の注意点は？

A4: これらの高度な手法をSAPに含める際は、その必要性、理論的背景、および解析結果の解釈方法を明確に記述することが重要です。特に規制当局への提出を伴う試験では、手法の妥当性や頑健性を裏付けるために、事前分布の選択根拠、感度分析の計画、そして計算方法（例：INLA）の詳細を具体的に示す必要があります。{{internal_link:ベイズ統計の導入と課題}}も確認しましょう。

Q5: 分布ラグ非線形モデル（DLNM）はどのようなデータに適用できますか？

A5: DLNMは、曝露とアウトカムが時間的に関連しており、かつ曝露の影響が遅れて現れる可能性のある時系列データに特に適しています。例えば、環境汚染物質（PM2.5、オゾンなど）と呼吸器・循環器疾患の関連、インフルエンザワクチン接種と発症の遅延効果、特定の薬剤投与と副作用発現までの潜伏期間、あるいは特定の食品摂取と疾患リスクの関係性など、幅広い分野の臨床研究や疫学研究に応用可能です。

まとめ

統計解析計画書（SAP）は、臨床研究の信頼性と科学的妥当性を保証するための「羅針盤」です。ICH E9ガイドラインに準拠し、主要評価項目から感度分析まで、すべての統計解析に関する意思決定を試験実施前に明確に記録することは、結果の客観性と再現性を高める上で不可欠です。本記事で紹介したように、R、ベイズ統計、INLA、そして分布ラグ非線形モデル（DLNM）といった最新のツールと手法をSAPに適切に組み込むことで、より深い洞察を得られるだけでなく、複雑な臨床的疑問にも統計的に頑健なアプローチで応えることが可能になります。
