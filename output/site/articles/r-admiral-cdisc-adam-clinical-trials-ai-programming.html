R, admiralで加速！臨床試験のCDISC ADaMデータ生成とAIプログラミング最前線

Photo from Unsplash

こんにちは、臨床統計ラボの管理人です。10年以上にわたり製薬企業やCROで臨床試験の統計解析に携わってきた経験を活かし、皆さんの臨床統計プログラミングスキル向上に貢献したいと思っています。

近年、R言語が臨床試験の分野で急速に存在感を高めています。特に、CDISC ADaMデータセットの作成を効率化するadmiralパッケージの登場は、臨床統計プログラマーにとって大きな変革をもたらしています。さらに、AIプログラミングの進化は、私たちの業務に新たな可能性を提示しつつあります。この記事では、Rとadmiralを活用したADaMデータ生成の具体的な方法から、最新の動向、そしてAIがもたらす未来について、実践的なコード例を交えながら深掘りしていきます。

この記事でわかること

  • Rとadmiralパッケージの基本的な使い方とそのメリット
  • CDISC ADaMデータセットの設計思想と、admiralを用いた効率的な生成方法
  • Rのrtablesパッケージによる臨床試験レポート作成の基礎
  • SASプログラミングとの比較で見るRの優位性と移行のポイント
  • AIプログラミングが臨床試験のデータ解析にもたらす可能性と最新動向

結論から言うと

臨床試験におけるデータ解析プログラミングは、R言語とadmiralパッケージの登場により、ますます効率的かつ標準化されてきています。特にCDISC ADaMデータセットの生成は、admiralを用いることでSASに匹敵する、あるいはそれ以上の生産性を実現可能になりました。そして、今後はAIプログラミングが、私たちの業務をさらに高度に自動化し、精度を高める重要な鍵となるでしょう。

R, admiral, CDISC, ADaM, 臨床試験, AIプログラミングとは？

このセクションでは、今回のテーマとなっている重要なキーワードについて、初心者の方にも分かりやすく解説していきます。

R言語：統計解析のデファクトスタンダード

Rは、統計計算とグラフィックスのための強力なオープンソースプログラミング言語です。学術界で広く利用されてきましたが、近年では製薬業界や金融業界など、様々な分野でデータ解析の主要ツールとして採用が進んでいます。豊富なパッケージ（拡張機能）エコシステムが特徴で、複雑な統計モデルの実装から、高品質なデータ可視化まで、あらゆる解析ニーズに対応できます。

admiral：ADaMデータセット生成の救世主

admiralは、RでCDISC ADaM（Analysis Data Model：解析用データセット）を効率的に作成するために開発されたパッケージ群です。CDISCの標準に準拠したADaMデータセットの変数導出や構造化を、tidyverse（Rのデータ操作パッケージ群）スタイルで直感的かつ体系的に行うことを可能にします。これにより、解析用データセット作成の生産性向上、コードの標準化、バリデーション作業の効率化が期待されています。

CDISC：臨床試験データの国際標準

CDISC (Clinical Data Interchange Standards Consortium) は、臨床試験データの収集、交換、提出に関する国際的な標準規格を策定している非営利団体です。SDTM (Study Data Tabulation Model：報告用データセット) やADaMなど、主要なデータモデルを提供しており、規制当局（FDAやPMDAなど）へのデータ提出において、これらの標準に準拠することが求められています。CDISC標準の採用は、データの相互運用性を高め、臨床試験の透明性と効率性を向上させるために不可欠です。

ADaM：解析に最適化されたデータモデル

ADaMは、CDISCが定めた解析用データセットの標準です。SDTMデータセットから、統計解析の目的（例：主要評価項目、副次評価項目）に合わせて変数を導出・加工し、解析プログラムが直接利用できる形式に整理されます。ADaMデータセットは、解析のトレーサビリティ（追跡可能性）を確保し、バリデーション（検証）を容易にすることを目的としています。主要なデータセットには、被験者レベルの解析情報を格納するADSL (Subject-Level Analysis Dataset) や、各評価項目に対応するADA_x_などが含まれます。

臨床試験：医薬品開発の要

臨床試験は、新しい医薬品や治療法の安全性（Safety）と有効性（Efficacy）を評価するために、人を対象に行われる研究です。第I相から第III相までの段階を経て、最終的に規制当局の承認を得ることを目指します。この過程で収集される膨大なデータは、統計的な手法を用いて解析され、医薬品の有効性を科学的に証明するための根拠となります。正確なデータ解析は、臨床試験の成功と患者さんへの安全な医薬品提供に直結します。

AIプログラミング：臨床統計の新たな地平

AIプログラミングとは、人工知能（AI）技術を活用して、プログラムコードの生成、最適化、デバッグなどを支援する取り組みを指します。最近注目されているGenerative AI（生成AI）は、自然言語での指示に基づいてコードを生成したり、既存のコードを改善したりすることが可能です。臨床試験のプログラミングにおいても、定型的なコードの生成や、複雑な統計解析コードの初期ドラフト作成、あるいはエラー箇所の特定などに活用されることで、プログラマーの生産性を劇的に向上させる可能性を秘めています。これは、RやSASといった既存のプログラミング環境をさらに強化する技術として期待されています。

統計的背景・理論

ADaMデータセットの設計は、統計解析の目的とトレーサビリティの確保という二つの柱に基づいています。解析プログラミングにおいて特に重要となるのは、ベースライン値からの変化量や、特定のイベント発生までの時間といった変数の導出です。ここでは、基本的な統計量の計算とADaMにおける考え方について解説します。

ベースラインからの変化量

多くの臨床試験では、治療効果を評価するために、治療開始前（ベースライン）と治療後における測定値の変化量を解析します。例えば、血圧や検査値、疾患スコアなどが対象となります。

変化量 (Change from Baseline) $\Delta y_i$ は、以下の数式で表されます。

$\Delta y_i = y_{i, \text{post}} - y_{i, \text{baseline}}$

ここで、$y_{i, \text{post}}$ は治療後の測定値、$y_{i, \text{baseline}}$ はベースライン測定値を示します。ADaMでは、このような変化量をCHGという変数名で格納することが一般的です。

イベント発生までの時間 (Time-to-Event)

がんの臨床試験などでは、特定のイベント（例：死亡、疾患の再発、治療中止）が発生するまでの期間を評価することがよくあります。これは生存時間解析（Survival Analysis）として知られています。イベント発生までの時間は、日数や月数で計算され、通常は治療開始日や初回投与日からイベント発生日までの差分として導出されます。

イベント発生までの期間 $T_i$ は、以下の数式で表されます。

$T_i = \text{EventDate}_i - \text{StartDate}_i$

ADaMでは、イベント発生までの期間（例：PFS (Progression-Free Survival) の期間）や、イベントの有無を示す変数（例：CNSR (Censor) フラグ）を格納します。admiralパッケージには、これらの変数導出をサポートする機能が豊富に用意されています。

SAS/Rコード実装

ここでは、Rのadmiralパッケージを中心に、CDISC ADaMデータセットを作成するコード例を示します。比較のために、簡単なSASコードも合わせて提示します。

まず、解析対象データセットの基本となるADSL（Subject-Level Analysis Dataset）の簡易版をRで作成します。これは通常、SDTMのDM（Demographics）ドメインやEX（Exposure）ドメインなどから作成されます。

# Rコード: admiralとtidyverseを使ったADaMデータセット生成

# 必要なパッケージを読み込みます
# install.packages(c("tidyverse", "admiral", "admiraldev", "rtables")) # 初回のみ実行
library(tidyverse)
library(admiral)
library(admiraldev)
library(rtables)

# -----------------------------------------------------------------------------
# 1. サンプルADSLデータセットの作成 (通常はSDTM DM/EXから変換)
# -----------------------------------------------------------------------------
# 架空の被験者レベル解析データセットADSLを生成します
adsl <- tibble::tribble(
  ~STUDYID, ~USUBJID, ~SITEID, ~SUBJID, ~TRT01A, ~TRT01AN, ~SEX, ~AGE, ~AGEU, ~RACE, ~SAFFL, ~RANDDT, ~TRTSDT, ~TRTEDT, ~DTHDT,
  "STUDY01", "001-001", "S01", "001", "Placebo", 0, "F", 35, "YEARS", "WHITE", "Y", as.Date("2023-01-05"), as.Date("2023-01-05"), as.Date("2023-06-30"), NA,
  "STUDY01", "001-002", "S01", "002", "Drug X", 1, "M", 42, "YEARS", "ASIAN", "Y", as.Date("2023-01-10"), as.Date("2023-01-10"), as.Date("2023-07-15"), NA,
  "STUDY01", "001-003", "S01", "003", "Drug X", 1, "F", 58, "YEARS", "WHITE", "Y", as.Date("2023-01-12"), as.Date("2023-01-12"), as.Date("2023-07-20"), as.Date("2023-07-01"),
  "STUDY01", "001-004", "S02", "004", "Placebo", 0, "M", 61, "YEARS", "BLACK", "Y", as.Date("2023-01-15"), as.Date("2023-01-15"), as.Date("2023-07-25"), NA,
  "STUDY01", "001-005", "S02", "005", "Drug X", 1, "F", 29, "YEARS", "WHITE", "Y", as.Date("2023-01-20"), as.Date("2023-01-20"), as.Date("2023-07-30"), NA
) %>%
  mutate(DTHFL = case_when(!is.na(DTHDT) ~ "Y", TRUE ~ "N")) # 死亡フラグ

# -----------------------------------------------------------------------------
# 2. admiralを用いたADSLの変数導出例
# -----------------------------------------------------------------------------
# 年齢グループ (AGEGR1) とベースライン体重 (BMIBL) を導出する例を想定。
# ここでは簡略化し、ADSLに直接BMIのベースライン値があるものとします。
# 実際のadmiralでは、ADSLにない変数を別のADaMからmergeしたり、SDTMから計算したりします。

adsl_derived <- adsl %>%
  # 年齢グループ変数の導出
  # admiraldev::derive_var_age_gr() を使用することもできますが、ここでは簡単な例を。
  mutate(
    AGEGR1 = case_when(
      AGE < 40 ~ "<40",
      AGE >= 40 & AGE < 60 ~ "40-59",
      AGE >= 60 ~ ">=60",
      TRUE ~ as.character(NA_real_)
    ),
    BMIBL = c(23.5, 26.1, 28.0, 24.9, 21.3) # 仮のベースラインBMI値
  ) %>%
  # 治療期間 (TRTDUR) を導出 (日数)
  derive_var_study_duration(dataset_name = "adsl_derived",
                            start_date = TRTSDT,
                            end_date = TRTEDT,
                            new_var = TRTDUR)

print(adsl_derived)

# -----------------------------------------------------------------------------
# 3. rtablesを用いた簡単なテーブル作成例 (患者背景表)
# -----------------------------------------------------------------------------
# rtablesは臨床試験のテーブル作成に特化したパッケージです。

# テーブル定義 (Table 1: 患者背景)
lyt <- basic_table() %>%
  split_cols_by("TRT01A") %>%
  add_overall_col("Overall") %>%
  # 年齢の要約統計量
  analyze("AGE", afun = function(x) list(
    "Mean (SD)" = rtables::r_stats_format(mean = mean(x, na.rm = TRUE), sd = sd(x, na.rm = TRUE), digits = 1),
    "Median (IQR)" = rtables::r_stats_format(median = median(x, na.rm = TRUE), iqr = IQR(x, na.rm = TRUE), digits = 1)
  ), table_names = "age_stats") %>%
  # 性別の度数と割合
  analyze("SEX", afun = function(x) {
    counts <- table(x)
    props <- prop.table(counts) * 100
    list(
      "Male" = paste0(counts["M"], " (", format(round(props["M"], 1), nsmall=1), "%)"),
      "Female" = paste0(counts["F"], " (", format(round(props["F"], 1), nsmall=1), "%)")
    )
  }, table_names = "sex_counts") %>%
  # 人種別の度数と割合
  analyze("RACE", afun = function(x) {
    counts <- table(x)
    props <- prop.table(counts) * 100
    list(
      "Asian" = paste0(counts["ASIAN"], " (", format(round(props["ASIAN"], 1), nsmall=1), "%)"),
      "White" = paste0(counts["WHITE"], " (", format(round(props["WHITE"], 1), nsmall=1), "%)"),
      "Black" = paste0(counts["BLACK"], " (", format(round(props["BLACK"], 1), nsmall=1), "%)")
    )
  }, table_names = "race_counts")

# テーブルの生成と表示
tbl <- build_table(lyt, adsl_derived)
print(tbl)

# テーブルをHTMLファイルとして保存 (必要に応じて)
# r2rtf::rtf_body(tbl, file = "patient_characteristics.rtf")

SASコード例

上記のRコードに対応する、基本的なSASのDATAステップとPROCステートメントの例です。SASでは、データステップで変数を作成し、PROC FREQやPROC MEANSで要約統計量や度数分布表を作成します。

/* SASコード: 基本的なADSL変数導出と要約統計量 */

/* --------------------------------------------------------------------------- */
/* 1. サンプルADSLデータセットの作成 (DATAステップ)                             */
/* --------------------------------------------------------------------------- */
data work.adsl;
  length STUDYID USUBJID SITEID SUBJID TRT01A SEX AGEU RACE SAFFL DTHFL $10;
  format RANDDT TRTSDT TRTEDT DTHDT yymmdd10.;
  input STUDYID $ USUBJID $ SITEID $ SUBJID $ TRT01A $ TRT01AN SEX $ AGE AGEU $ RACE $ SAFFL $ RANDDT :yymmdd. TRTSDT :yymmdd. TRTEDT :yymmdd. DTHDT :yymmdd.;
  datalines;
STUDY01 001-001 S01 001 Placebo 0 F 35 YEARS WHITE Y 2023-01-05 2023-01-05 2023-06-30 . 
STUDY01 001-002 S01 002 DrugX   1 M 42 YEARS ASIAN Y 2023-01-10 2023-01-10 2023-07-15 . 
STUDY01 001-003 S01 003 DrugX   1 F 58 YEARS WHITE Y 2023-01-12 2023-01-12 2023-07-20 2023-07-01
STUDY01 001-004 S02 004 Placebo 0 M 61 YEARS BLACK Y 2023-01-15 2023-01-15 2023-07-25 . 
STUDY01 001-005 S02 005 DrugX   1 F 29 YEARS WHITE Y 2023-01-20 2023-01-20 2023-07-30 . 
;

/* 死亡フラグの導出 */
data work.adsl_derived;
  set work.adsl;
  if DTHDT ne . then DTHFL = 'Y';
  else DTHFL = 'N';

  /* 年齢グループの導出 */
  if AGE < 40 then AGEGR1 = '<40';
  else if AGE >= 40 and AGE < 60 then AGEGR1 = '40-59';
  else if AGE >= 60 then AGEGR1 = '>=60';
  else AGEGR1 = ' '; /* 欠損値の場合 */

  /* 仮のベースラインBMI値 */
  /* SASのDATAステップで配列を使う例は複雑になるため、ここでは簡略化 */
  /* 実際には、SDTMのVS（Vital Signs）などから計算して結合します */
  if USUBJID = '001-001' then BMIBL = 23.5;
  else if USUBJID = '001-002' then BMIBL = 26.1;
  else if USUBJID = '001-003' then BMIBL = 28.0;
  else if USUBJID = '001-004' then BMIBL = 24.9;
  else if USUBJID = '001-005' then BMIBL = 21.3;

  /* 治療期間 (日数) の導出 */
  TRTDUR = TRTEDT - TRTSDT;
run;

proc print data=work.adsl_derived(obs=5);
  title 'Derived ADSL Dataset (SAS)';
run;

/* --------------------------------------------------------------------------- */
/* 2. SASを用いた簡単なテーブル作成例 (患者背景表)                               */
/* --------------------------------------------------------------------------- */
/* 年齢の要約統計量 */
proc means data=work.adsl_derived n mean std median q1 q3;
  class TRT01A;
  var AGE;
  output out=work.age_stats mean=Mean std=Std median=Median q1=Q1 q3=Q3;
run;

/* 性別、人種別の度数と割合 */
proc freq data=work.adsl_derived;
  tables TRT01A * (SEX RACE) / nopercent norow nocol;
run;

Rのadmiralパッケージは、特にCDISC ADaMの標準に合わせた変数導出関数（derive_var_xxxのような関数）が豊富に用意されているため、データセット作成のロジックを標準化し、コードの再利用性を高めることができます。一方、SASはDATAステップによる柔軟なデータ操作と、PROC MEANS, PROC FREQなどの強力なプロシージャで、長年の実績と信頼があります。しかし、admiralはtidyverseのパイプ処理と組み合わせることで、より宣言的（Declarative）かつ読みやすいコードを書けるというメリットがあります。

実務での活用場面

Rとadmiralパッケージは、臨床試験のデータ解析プログラミングにおいて多岐にわたる場面で活用できます。

1. CDISC ADaMデータセットの効率的な作成

最も主要な活用場面です。SDTMデータセットから、年齢層、BMI、ベースラインからの変化量、イベント発生までの期間など、解析に必要な変数をCDISC ADaMの標準に準拠して導出します。admiralの各種関数を使用することで、手作業でのコーディングミスを減らし、バリデーション作業を効率化できます。これにより、統計解析報告書（CSR: Clinical Study Report）作成のリードタイム短縮に貢献します。

2. 解析コードの標準化と再利用

admiralは、CDISCのADaM実装ガイド（ADaM IG）に沿った変数導出ロジックを提供するため、プロジェクト間でのコードの標準化を促進します。一度作成したADaM生成用のスクリプトは、異なる臨床試験データに対しても容易に適用でき、解析コードの再利用性を大幅に向上させます。これにより、複数試験を横断した解析（統合解析など）の準備も効率的に行えます。

3. テーブル・リスト・フィギュア (TLF) の自動生成

rtablesやTplyr, ternなどのRパッケージは、臨床試験の報告書で求められる標準的なテーブル（例：患者背景表、有効性評価項目サマリー表）やリスト、フィギュアを自動生成する強力な機能を提供します。これらのパッケージをadmiralで作成したADaMデータセットと組み合わせることで、プログラミングからアウトプット生成までの一貫したワークフローをRで構築できます。これにより、手作業でのテーブル作成に伴うエラーリスクを低減し、品質向上と効率化を図ることができます。

4. 探索的データ解析 (EDA) と統計モデリング

Rは、データ探索や高度な統計モデリングに非常に強みを持っています。ADaMデータセットが整備されていれば、ggplot2を使った美しい可視化や、lme4による混合モデル、survivalパッケージによる生存時間解析など、幅広い統計解析をスムーズに実施できます。これにより、臨床試験のデータからより深い洞察を得ることが可能になります。

5. AIプログラミングとの連携

AIプログラミング、特にGenerative AIの登場は、Rでの臨床試験プログラミングにも大きな影響を与え始めています。例えば、自然言語で「患者背景表を作成するRコードを生成して」と指示するだけで、rtablesを使ったコードのドラフトが生成されるといったことが現実になりつつあります。また、既存のRコードのデバッグ支援や、最適化の提案などもAIによって行われる可能性があります。これにより、プログラマーはより複雑で創造的なタスクに集中できるようになるでしょう。

海外の最新動向

Rの臨床試験分野での活用は、海外の主要なコミュニティで活発に議論され、その動きは加速しています。

Rification of Clinical Trials

PHUSE (Pharmaceutical Users Software Exchange)、PSI (Statisticians in the Pharmaceutical Industry)、ASA Biopharmaceutical Section (American Statistical Association) などの主要な国際コミュニティでは、「Rification of Clinical Trials」（臨床試験のR化）と称して、Rを臨床試験の規制当局提出データ・解析に活用するための議論やプロジェクトが進められています。これらのコミュニティは、Rのパッケージ開発、バリデーション、トレーニングなどに関するワーキンググループを立ち上げ、ガイダンスの策定やベストプラクティスの共有を行っています。

特にPHUSEは、Rを用いたCDISC標準データセット（SDTM, ADaM）のパイロットプロジェクトを実施しており、admiralを含む{pharmaverse}のエコシステムがその中核を担っています。これにより、Rが規制当局提出データ生成の主要ツールとなり得ることを実証しようとしています。

  • PHUSE R Validation Hub: https://www.phuse.global/RValidationHub
  • {pharmaverse}: https://pharmaverse.org/

admiralパッケージの進化とコミュニティ

admiralパッケージは、{pharmaverse}プロジェクトの一部として継続的に開発が進められています。定期的なアップデートが行われ、新たな機能が追加されるとともに、コミュニティからのフィードバックに基づいて改善されています。また、admiralの設計思想は、CDISCのADaM IG (Implementation Guide) に厳密に準拠しており、規制当局への提出資料の信頼性を高めることを目指しています。活発なGitHubリポジトリと議論フォーラムがあり、ユーザーは開発チームや他のユーザーと情報交換を行うことができます。

  • admiral GitHub: https://github.com/pharmaverse/admiral

AIプログラミングと規制当局の視点

AIプログラミング、特にGenerative AIの活用については、まだ規制当局からの明確なガイダンスは出ていません。しかし、FDA（米国食品医薬品局）などは、医薬品開発におけるAI/MLの利用に関する議論を進めており、その品質管理やバリデーションのあり方が焦点となっています。現時点では、AIが生成したコードは最終的な解析コードとしてそのまま利用するのではなく、プログラマーがレビューし、バリデーションを行うことが不可欠です。将来的には、AIが生成したコードの品質保証メカニズムが確立されれば、臨床試験の解析プロセスにおいてより広範に導入される可能性があります。

よくある質問（FAQ）

Q1: admiralパッケージはSASプログラミングの完全に置き換えになりますか？

A1: 現時点では「完全に置き換わる」とまでは言い切れませんが、CDISC ADaMデータセットの生成に関しては、admiralがSASと同等、またはそれ以上の効率性と標準化を提供できる場面が増えています。特に、tidyverseの思想に基づいた宣言的なコーディングスタイルは、可読性とメンテナンス性を向上させます。SASには長年の実績と膨大な既存コードベースがあり、すぐに全てが置き換わるわけではありませんが、新規プロジェクトや特定の解析タスクではRとadmiralの採用が加速するでしょう。

Q2: Rで作成したADaMデータセットや解析結果は、規制当局（FDA/PMDA）に提出できますか？

A2: はい、可能です。FDAやPMDAは特定のソフトウェアに限定しておらず、重要なのは提出されたデータや解析結果の品質、トレーサビリティ（追跡可能性）、そしてバリデーション（検証可能性）です。Rのパッケージ（admiral、rtablesなど）は、これらの要件を満たすように設計・開発されています。重要なのは、適切なバリデーションプロセスと品質管理システムを構築し、それに基づいてRコードを実行し、結果を文書化することです。PHUSEなどのコミュニティが、Rの規制当局提出に向けたガイダンス作成を積極的に進めています。

Q3: Rの学習コストは高いですか？

A3: SASに慣れている方にとっては、最初は戸惑うこともあるかもしれません。しかし、tidyverseのエコシステムが提供する一貫した文法と直感的な関数群は、学習曲線を大幅に緩やかにしています。特に、データフレーム操作の概念はSASのDATAステップと類似しているため、比較的スムーズに移行できる部分も多いでしょう。一度基本を習得すれば、Rの持つ強力な機能と豊富なパッケージ群が、あなたのプログラミングの幅を大きく広げてくれます。オンラインの学習リソースも非常に豊富です。

Q4: AIプログラミングは臨床試験のプログラマーの仕事を奪いますか？

A4: いいえ、むしろプログラマーの役割を進化させ、より戦略的で高度な業務に集中できるようになるでしょう。AIは定型的なコード生成や初歩的なデバッグを支援することで、プログラマーが繰り返し行う作業の負担を軽減します。これにより、プログラマーは複雑な解析ロジックの設計、結果の解釈、バリデーション戦略の考案、そして新しい統計手法の導入といった、より付加価値の高い業務に時間を費やせるようになります。AIは強力な「副操縦士」として、私たちの生産性を高めるツールと捉えるべきです。

この記事では、R言語とadmiralパッケージが、臨床試験におけるCDISC ADaMデータセット生成と解析プログラミングに革命をもたらしている現状を解説しました。admiralは、CDISC標準に準拠したADaM作成を効率化し、rtablesなどのパッケージと連携することで、一貫したRベースの解析ワークフローを構築可能です。そして、AIプログラミングの進化は、このワークフローをさらに強化し、将来的にプログラマーの生産性と解析の精度を飛躍的に向上させる可能性を秘めています。
