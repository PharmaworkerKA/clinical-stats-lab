標本中央値と偏り：歪んだ分布における臨床統計の注意点

結論から言うと

歪んだ分布を示す臨床試験データでは、外れ値の影響を受けにくい標本中央値が、標本平均値よりもロバストな代表値となります。しかし、標本中央値は小規模なサンプルサイズや歪みが大きい分布では、真の母中央値に対する偏り（バイアス）を持つことが知られています。この偏りの存在を理解し、その統計的特性を踏まえた上で適切な解析戦略を立てることが、生物統計 の観点から極めて重要です。本記事では、この標本中央値の特性と実務での活用場面、そしてSAS/Rによる実践的な解析手法を詳しく解説し、日本の臨床統計担当者の皆様が正確なデータ解釈を行えるよう支援します。

標本中央値とは？

まず、標本中央値（Sample Median）が一体何であるか、その基本的な概念から確認していきましょう。

標本中央値とは、観測されたデータ（標本）を小さい順（または大きい順）に並べたときに、ちょうど真ん中に位置する値のことです。例えば、5つのデータ「10, 20, 30, 40, 100」があった場合、これらを並べ替えて「10, 20, 30, 40, 100」となり、真ん中の値である「30」が標本中央値となります。データ数が偶数の場合は、真ん中に位置する2つの値の平均を中央値とします。例えば「10, 20, 30, 40」の場合、20と30の平均である25が標本中央値です。

これに対し、最も一般的に用いられる代表値である 標本平均値（Sample Mean）は、全てのデータを合計し、その個数で割った値です。上記の例「10, 20, 30, 40, 100」の場合、標本平均値は (10+20+30+40+100) / 5 = 200 / 5 = 40 となります。この例でわかるように、データの中に「100」のような極端に大きな値（外れ値）があると、標本平均値は大きく影響を受け、データの中心を適切に表せなくなることがあります。しかし、標本中央値は外れ値の影響を受けにくく、データの頑健な（ロバストな）代表値として機能します。この特性から、歪んだ分布 のデータや外れ値を含むデータでは、標本中央値が好まれることがあります。

臨床試験の分野では、患者さんの痛みスコア（VASスケール）、QOL（生活の質）評価、特定の薬物濃度、血中マーカーの測定値など、様々なデータが必ずしも左右対称の正規分布を示すわけではありません。むしろ、多くの場合で歪んだ分布を示すことが一般的です。このような状況で、データの中心傾向を適切に把握するために、標本中央値の理解と活用は不可欠となります。

統計的背景・理論

標本中央値が臨床統計においてなぜ重要であり、またどのような統計的課題を抱えているのかを、偏り（Bias）の概念と合わせて掘り下げていきましょう。

不偏性（Unbiasedness）と偏り（Bias）

統計的推定において、不偏性（Unbiasedness）とは、ある統計量（推定量）の期待値（E[・]）が、推定したい真の母数（母集団の特性値）に等しい性質を指します。例えば、標本平均 $\bar{X}$ は母平均 $\mu$ の不偏推定量です。これは、$E[\bar{X}] = \mu$ が成り立つことを意味します。つまり、同じ母集団から何度も標本を抽出し、そのたびに標本平均を計算すると、その標本平均たちの平均は真の母平均に近づくということです。

一方、偏り（Bias）は、推定量の期待値と真の母数との差を示します。$\text{Bias}(\hat{\theta}) = E[\hat{\theta}] - \theta$ と表され、この差が0でない場合に「偏りのある（biased）推定量」とみなされます。

標本中央値の不偏性：対称分布と歪んだ分布

標本中央値は、分布が左右対称である場合には母中央値の不偏推定量となります。例えば、正規分布（ベル型曲線を描く左右対称の分布）のようなケースでは、母平均、母中央値、最頻値がすべて一致するため、標本中央値も母中央値（および母平均）の不偏推定量として機能します。

しかし、現実の臨床試験データでは、左右対称でない 歪んだ分布（Skewed Distribution）が頻繁に現れます。歪んだ分布には、大きく分けて以下の2種類があります。

  • 正の歪み（右に裾が長い分布、Right Skewed Distribution）：データが小さい値に集中し、大きな値に少数の外れ値が広がる分布。平均値 > 中央値 > 最頻値となる傾向があります。例：薬物濃度、所得、一部の疾患の症状スコア。
  • 負の歪み（左に裾が長い分布、Left Skewed Distribution）：データが大きい値に集中し、小さな値に少数の外れ値が広がる分布。平均値 < 中央値 < 最頻値となる傾向があります。例：試験の成績（多くの人が高得点）、反応時間データ。

このような歪んだ分布の場合、標本中央値は一般的に母中央値の不偏推定量ではありません。特にサンプルサイズが小さい場合、その偏りは無視できないものとなることがあります。例えば、正の歪みを持つ分布から標本を抽出すると、標本中央値は真の母中央値よりもわずかに小さくなる傾向があります（負の偏り）。これは、小さい値にデータが集中し、大きい値側の裾が長く伸びる特性が、サンプリング時に中央値の計算に影響を与えるためです。

母中央値 $M$ は、確率変数 $X$ について $P(X \le M) = 0.5$ を満たす値として定義されます。標本中央値 $\tilde{X}$ は、順序統計量 $X_{(k)}$ を用いて、サンプルサイズ $n$ が奇数の場合は $X_{((n+1)/2)}$、偶数の場合は $(X_{(n/2)} + X_{(n/2+1)})/2$ と定義されます。歪んだ分布における標本中央値の期待値 $E[\tilde{X}]$ は、一般に母中央値 $M$ とは一致せず、$E[\tilde{X}] \neq M$ となるため、偏りが発生するのです。

この偏りは、特にデータが極端に歪んでいる場合や、サンプルサイズが非常に小さい場合に顕著になります。臨床試験では、希少疾患の試験や探索的な早期相試験でサンプルサイズが限られることが多いため、標本中央値を用いる際にはこの偏りに十分注意を払う必要があります。

SAS/Rコード実装

ここでは、指数分布（代表的な正に歪んだ分布）からデータをシミュレーション抽出し、標本中央値がどのように偏りを持つかを確認するSASおよびRのコード例をご紹介します。このシミュレーションを通じて、標本中央値 の統計的挙動を直感的に理解しましょう。

サンプルデータ生成とシミュレーションの考え方

母集団として、尺度パラメータ $\lambda=1$ の指数分布（Exponential distribution）を仮定します。この分布の母中央値は $\text{ln}(2) \cdot \lambda \approx 0.693$ です。私たちはこの母集団から小さな標本（例えば $n=5$ や $n=10$）を繰り返し抽出し、各標本の中央値を計算します。これらの標本中央値の平均値が、真の母中央値からどれだけ離れているかを見ることで、偏りの程度を評価します。

SASコードによるシミュレーション

SASでは PROC IML を用いることで、効率的にシミュレーションを実行できます。PROC IML は行列言語であり、ループ処理や統計関数の適用に優れています。

/* ------------------------------------------------------------------ */
/* SASコード：標本中央値の偏りシミュレーション (指数分布)           */
/* ------------------------------------------------------------------ */

/* 1. PROC IML を用いたシミュレーション */
PROC IML;
  /* シミュレーション設定 */
  call randseed(12345); /* 乱数シードの固定 */
  lambda = 1;          /* 指数分布の尺度パラメータ */
  true_median = log(2) * lambda; /* 母中央値の計算 */
  print "真の母中央値 = " true_median;

  num_simulations = 10000; /* シミュレーション繰り返し回数 */
  sample_size_small = 5;   /* 小さいサンプルサイズ */
  sample_size_large = 100; /* 大きいサンプルサイズ */

  /* 結果を格納する行列 */
  median_small = j(num_simulations, 1, .);
  median_large = j(num_simulations, 1, .);

  /* シミュレーションループ */
  do i = 1 to num_simulations;
    /* 小さいサンプルサイズで指数分布データを生成 */
    data_small = rand("EXPONENTIAL", lambda, sample_size_small, 1);
    median_small[i] = quantile(data_small, 0.5); /* 標本中央値を計算 */

    /* 大きいサンプルサイズで指数分布データを生成 */
    data_large = rand("EXPONENTIAL", lambda, sample_size_large, 1);
    median_large[i] = quantile(data_large, 0.5); /* 標本中央値を計算 */
  end;

  /* 結果の集計と表示 */
  mean_median_small = mean(median_small);
  bias_small = mean_median_small - true_median;
  print "\n--- サンプルサイズ n=" sample_size_small " ---";
  print "シミュレートされた標本中央値の平均 = " mean_median_small;
  print "偏り (Bias) = " bias_small;

  mean_median_large = mean(median_large);
  bias_large = mean_median_large - true_median;
  print "\n--- サンプルサイズ n=" sample_size_large " ---";
  print "シミュレートされた標本中央値の平均 = " mean_median_large;
  print "偏り (Bias) = " bias_large;
QUIT;

/* 簡略化のため、ここではIMLのprint出力で結果を確認 */
/* より本格的な可視化はRコードで示します */

SASコードの解説:

  • PROC IML;: 行列言語モードに入ります。
  • call randseed(12345);: 乱数シードを固定することで、常に同じ結果を再現できるようにします。生物統計 におけるシミュレーションでは再現性が重要です。
  • lambda = 1; true_median = log(2) * lambda;: 指数分布のパラメータと真の母中央値を定義します。log関数は自然対数を計算します。
  • num_simulations: シミュレーションを10,000回繰り返します。これにより、標本中央値の期待値をより正確に推定できます。
  • sample_size_small, sample_size_large: 比較のため、小さいサンプルサイズと大きいサンプルサイズを設定します。
  • rand("EXPONENTIAL", lambda, sample_size, 1);: 指数分布に従う乱数を指定されたサンプルサイズで生成します。
  • quantile(data, 0.5);: データの50パーセンタイル、すなわち標本中央値を計算します。SASの quantile 関数はパーセンタイルを計算するのに便利です。
  • mean(median_small);: 10,000回のシミュレーションで得られた標本中央値の平均を計算します。これが標本中央値の期待値の推定値となります。
  • bias_small = mean_median_small - true_median;: シミュレーションから得られた標本中央値の平均と真の母中央値との差が「偏り」です。

このSASコードを実行すると、小さいサンプルサイズ ($n=5$) では標本中央値の平均が真の母中央値 ($0.693$) よりも小さくなる（負の偏りがある）ことが確認できます。一方、大きいサンプルサイズ ($n=100$) では、偏りが大幅に減少していることがわかるでしょう。これは、標本中央値 の漸近的不偏性を示唆しています。

Rコードによるシミュレーションと可視化

Rでは tidyverse パッケージ群を使用し、データ操作と可視化を効率的に行います。purrr パッケージで繰り返し処理を行い、ggplot2 で結果をグラフ化します。

# ------------------------------------------------------------------ #
# Rコード：標本中央値の偏りシミュレーション (指数分布)               #
# ------------------------------------------------------------------ #

# 必要なパッケージをロード
library(tidyverse)

# シミュレーション設定
set.seed(12345) # 乱数シードの固定
lambda <- 1     # 指数分布の尺度パラメータ
true_median <- log(2) * lambda # 母中央値の計算
cat("真の母中央値 = ", true_median, "\n\n")

num_simulations <- 10000 # シミュレーション繰り返し回数
sample_sizes <- c(5, 10, 100) # 異なるサンプルサイズ

# シミュレーション関数の定義
run_simulation <- function(n_samples) {
  # 指定されたサンプルサイズでデータを生成し、標本中央値を計算する関数
  replicate(num_simulations, {
    data_exp <- rexp(n_samples, rate = 1/lambda) # 指数分布データを生成 (rate = 1/lambda)
    median(data_exp) # 標本中央値を計算
  })
}

# 各サンプルサイズでシミュレーションを実行
sim_results <- map_dfr(sample_sizes, ~ {
  medians <- run_simulation(.x)
  tibble(
    sample_size = .x,
    simulated_median = medians
  )
}) %>%
  # 各シミュレーション結果の要約を計算
  group_by(sample_size) %>%
  summarise(
    mean_sim_median = mean(simulated_median),
    bias = mean_sim_median - true_median,
    .groups = 'drop'
  )

print(sim_results)

# シミュレーション結果の可視化
# 各サンプルサイズにおける標本中央値の分布を表示
plot_data <- map_dfr(sample_sizes, ~ {
  medians <- run_simulation(.x)
  tibble(
    sample_size = factor(.x), # グラフ表示用に因子型に変換
    simulated_median = medians
  )
})

ggplot(plot_data, aes(x = simulated_median, fill = sample_size)) +
  geom_density(alpha = 0.6) +
  geom_vline(aes(xintercept = true_median), linetype = "dashed", color = "red", size = 1) +
  facet_wrap(~ sample_size, scales = "free_y") +
  labs(
    title = "指数分布における標本中央値の分布と偏り",
    subtitle = paste0("真の母中央値 = ", round(true_median, 3)),
    x = "シミュレートされた標本中央値",
    y = "密度",
    fill = "サンプルサイズ"
  ) +
  theme_minimal() +
  theme(legend.position = "bottom")

# admiral/rtablesの活用例 (記述統計表の作成)
# 例えば、架空の臨床試験データで治療群ごとの中央値を記述統計表として出力する
# まず、架空の臨床試験データを生成
set.seed(42)
patient_data <- tibble(
  USUBJID = paste0("PAT", 1:100),
  TRT01A = sample(c("Placebo", "Drug A", "Drug B"), 100, replace = TRUE, prob = c(0.3, 0.4, 0.3)),
  AVAL = c(rexp(30, rate = 1/1), # Placebo: lambda=1
           rexp(40, rate = 1/0.8), # Drug A: lambda=0.8 (slightly lower median)
           rexp(30, rate = 1/0.9)) # Drug B: lambda=0.9
) %>%
  mutate(AVAL = round(AVAL, 1))

# rtablesを使って記述統計表を作成
# rtablesはclinical trial reportingに適したテーブルを生成します
library(rtables)

# 統計量の計算関数
median_sd_n <- function(x) {
  list(Median = median(x, na.rm = TRUE),
       SD = sd(x, na.rm = TRUE),
       N = length(x[!is.na(x)]))
}

# rtablesのレイアウト定義
tbl <- basic_table() %>%
  split_cols_by("TRT01A") %>%
  add_colcounts() %>%
  analyze(vars = "AVAL",
          afun = median_sd_n,
          format = "xx.x (xx.x) [xx]", # Median (SD) [N]
          table_names = "aval_stats")

# テーブルの生成
result_table <- build_table(tbl, patient_data)

# テーブル表示
result_table

Rコードの解説:

  • library(tidyverse): dplyr, ggplot2, purrr などの主要パッケージをまとめてロードします。
  • set.seed(12345): SASと同様に乱数シードを固定し、再現性を確保します。
  • true_median <- log(2) * lambda: 真の母中央値を計算します。
  • run_simulation 関数: 指定されたサンプルサイズで num_simulations 回、指数分布からデータを抽出し、各標本の標本中央値を計算するヘルパー関数です。
  • map_dfr(): purrr パッケージの関数で、sample_sizes の各要素に対して run_simulation を実行し、結果をデータフレームとして結合します。tidyverse スタイルのデータ処理の強力な例です。
  • group_by() %>% summarise(): シミュレーション結果をサンプルサイズごとに集計し、標本中央値の平均値と偏りを計算します。
  • ggplot(): ggplot2 を使って、各サンプルサイズでの標本中央値の分布を密度プロットで可視化します。赤い破線が真の母中央値を示しており、小さいサンプルサイズほど分布の中心が赤い破線からずれている（偏りがある）ことが視覚的に確認できます。
  • admiral/rtablesの活用例: 臨床試験の報告書作成に特化したパッケージ rtables を用いて、治療群ごとの記述統計表（中央値、標準偏差、N数）を作成する例を示します。これにより、実際の臨床統計 レポートでどのように中央値情報が提示されるかを示唆します。basic_table(), split_cols_by(), analyze() といった関数をパイプ (%>%) でつなげて、柔軟にテーブルレイアウトを定義できるのが特徴です。

これらのコードを実行することで、サンプルサイズが小さいほど標本中央値に偏りが見られ、サンプルサイズが大きくなると偏りが減少していく、という統計的性質を実感できます。

実務での活用場面

臨床統計 の現場では、標本中央値 が以下のような場面で特に有用となります。

  1. 歪んだ分布を示すエンドポイントの解析: 臨床試験で測定される多くのバイオマーカー（例：CRP、肝酵素、サイトカインなど）や、患者報告アウトカム（PROs; Patient-Reported Outcomes）である痛みスコア（Visual Analog Scale: VAS）、QOL（Quality of Life）スコアなどは、しばしば正規分布から大きく逸脱し、歪んだ分布 を示します。このようなデータに対しては、外れ値の影響を受けやすい標本平均値よりも、標本中央値の方がデータの中心傾向をより適切に代表する指標となります。

    • 具体例: 潰瘍性大腸炎の臨床試験で、疾患活動性スコア（Mayo Scoreなど）の改善度を評価する場合。多くの患者が軽度から中等度の改善を示す一方で、一部の患者が劇的な改善を示すような分布であれば、中央値が全体の傾向をよりよく表す可能性があります。

  2. 生存時間解析: 抗がん剤の臨床試験などで主要評価項目となる「無増悪生存期間（Progression-Free Survival, PFS）」や「全生存期間（Overall Survival, OS）」などの生存時間データは、その性質上、右に大きく歪んだ分布を示します。これらのデータでは、カプラン・マイヤー曲線から推定される「中央生存期間（Median Survival Time）」が治療効果の主要な指標として広く用いられます。これは本質的に、生存時間の中央値を推定していることになります。{{internal_link:生存時間解析の基礎}}のような詳細なトピックも参照してください。

  3. 薬物動態（PK）パラメータの記述: 薬物の血中濃度推移から算出される最高血中濃度（Cmax）や血中濃度曲線下面積（AUC）などの薬物動態パラメータも、個体差が大きいためしばしば歪んだ分布を示します。特に、治療薬物モニタリング（TDM）の目的で個々の患者の曝露量を評価する際には、中央値と範囲（最小値、最大値、四分位範囲など）が、その変動性をより適切に記述するために用いられます。

  4. 小規模な探索的臨床試験: 希少疾患を対象とした試験や、早期相（Phase I/IIa）の臨床試験では、被験者数が限られることが一般的です。このような小規模データでは、たった一つの外れ値が標本平均値に与える影響が大きくなるため、データの安定した代表値として標本中央値が選択されることがあります。ただし、前述の通り、小規模なデータでは標本中央値自体も偏りを持つ可能性があるため、その解釈には慎重さが求められます。

  5. 規制当局への提出資料: FDAやPMDAなどの規制当局は、主要評価項目の解析において、適切な統計量の選択を求めています。歪んだ分布の場合には、標本中央値を用いた解析結果が受け入れられるケースも増えてきています。特に、患者報告アウトカムや一部の生物学的エンドポイントにおいては、中央値が主要な記述統計量または効果量として採用されることがあります。比較表として、平均値と中央値の特性を整理しておきましょう。

特性 標本平均値（Sample Mean） 標本中央値（Sample Median）
計算方法 全てのデータの合計を個数で割る データを並べたときの中央の値
外れ値の影響 非常に受けやすい（Sensitive） 受けにくい（Robust）
分布の形状 正規分布（左右対称）で最適 歪んだ分布で特に有用
不偏性（対称分布） 不偏推定量である 不偏推定量である
不偏性（歪んだ分布） 不偏推定量である（大きなサンプルサイズで） 一般に偏りを持つ（特に小規模サンプルで）
適用場面 連続変数、正規分布に近いデータ 順序尺度データ、歪んだ分布、外れ値が多いデータ

このように、標本中央値は多くの臨床試験の場面で、非常に強力なツールとなり得ますが、その統計的特性、特に偏りを理解した上で、適切に使いこなすことが生物統計 のプロフェッショナルには求められます。

海外の最新動向

臨床統計 の分野では、国際的な会議やガイドラインを通じて、統計解析手法の標準化と進化が常に議論されています。標本中央値 の役割と適切な利用についても、PSI (Statisticians in the Pharmaceutical Industry)、ASA (American Statistical Association)、PHUSE (Pharmaceutical Users Software Exchange) といった主要な国際コミュニティで活発な議論が交わされています。

ICH E9(R1) 統計的原則への追補：Estimandsと感度分析

特に注目すべきは、ICH E9 Addendum on Estimands and Sensitivity Analysis in Clinical Trials (R1) です。これは2019年に最終化されたガイドラインであり、臨床試験の主要評価項目（Primary Endpoint）を定義する上で、何が推定対象であるか（Estimand）を明確にすることの重要性を強調しています。このガイドラインでは、連続変数の集団レベルの要約として、平均値だけでなく、中央値 やその他の分位点も推定の対象となりうると明記されています。

  • ICH E9(R1) Draft (2019) のセクション3.2.1では、「For a continuous variable, examples of population-level summaries include the mean, median, standard deviation, and quantiles.」と述べられています。これは、平均値だけが主要な統計量ではないという認識が国際的に広がっていることを示しており、歪んだ分布 のデータに対して中央値を用いる解析の正当性を裏付けるものです。
  • このガイダンスは、日本のPMDAを含む各国の規制当局が採用しているため、日本の臨床統計担当者も、中央値をEstimandの一部として適切に定義し、解析計画書（SAP; Statistical Analysis Plan）に明記する重要性が高まっています。

FDA/EMAのガイダンスと患者報告アウトカム（PRO）

FDA（米国食品医薬品局）やEMA（欧州医薬品庁）の統計関連ガイダンスでも、特定の疾患領域や評価項目において、中央値が重要な記述統計量として、あるいは主要評価項目の一部として受け入れられる事例が増えています。

  • 患者報告アウトカム (PRO): 患者さんの主観的な症状（痛み、倦怠感、QOLなど）を評価するPROデータは、しばしば歪んだ分布を示します。FDAやEMAは、PROデータを主要評価項目とする場合、中央値や分位点を用いた解析を推奨することがあります。これは、患者さんの体験をより正確に反映する指標として、中央値が平均値よりも優れていると判断されるためです。
    • 例: 慢性疼痛治療薬の臨床試験において、患者さんが報告する痛みの強さの改善度を評価する際に、中央値の変化が主要な効果指標となることがあります。
  • 希少疾患や小児科領域: サンプルサイズが非常に小さいこれらの領域の臨床試験では、平均値が外れ値によって大きく変動するリスクがあるため、中央値のようなロバストな統計量が重視される傾向にあります。

よくある質問（FAQ）

Q1: 標本中央値と標本平均、どちらを使うべきですか？

A1: データが正規分布に近い左右対称な分布を示す場合は、標本平均が一般的に効率的な推定量であり、好まれます。しかし、データが歪んだ分布 を示す場合や、著しい外れ値を含む場合は、外れ値の影響を受けにくい標本中央値 の方が、データの中心傾向をより適切に表現できます。どちらを選ぶべきかは、データの特性と解析の目的に応じて慎重に判断する必要があります。両方の統計量を報告し、比較することも有効なアプローチです。

Q2: 標本中央値の偏りは、どの程度問題になりますか？

A2: 標本中央値の偏りは、主にサンプルサイズが小さい場合や、分布の歪みが大きい場合に顕著になります。サンプルサイズが十分に大きくなると、標本中央値は漸近的に不偏性を持つため、偏りの問題は小さくなります。しかし、小規模な臨床試験では、この偏りが結果の解釈に影響を与える可能性があるため、注意が必要です。シミュレーションやブートストラップ法などを活用して、偏りの程度を評価することも検討されます。

Q3: 小規模臨床試験で中央値を使う際の注意点は？

A3: 小規模臨床試験で中央値を使用する場合、偏り の問題に加えて、推定量の分散が大きい（精度が低い）可能性がある点に注意が必要です。サンプルサイズが小さいと、標本中央値のばらつきが大きくなり、真の母中央値からの信頼区間が広くなる傾向があります。解析計画書（SAP）には、中央値を選択した理由と、その統計的特性（偏りの可能性など）に関する考慮事項を明確に記述することが重要です。{{internal_link:臨床試験のサンプルサイズ計算}}に関する知識も役立つでしょう。

Q4: 歪んだ分布でも不偏な中央値推定方法はありますか？

A4: 歪んだ分布であっても、ブートストラップ法（Bootstrap Method）などの再サンプリング手法を用いることで、標本中央値の信頼区間を構築したり、偏りを補正したりすることが可能です。また、分位点回帰（Quantile Regression）のような手法は、分布全体の中央値や他の分位点に対する共変量の影響を直接モデル化できるため、平均値の回帰分析が不適切となる歪んだ分布データに有効です。これらの手法は、生物統計 の高度な解析テクニックとして知られています。

まとめ

本記事では、標本中央値 の概念から始まり、その統計的特性、特に歪んだ分布 における偏り の重要性について深く掘り下げてきました。生物統計 および臨床統計 の専門家として、私は以下の点を強調したいと思います。

  1. 標本中央値のロバスト性: 外れ値や歪んだ分布に対して、標本中央値は標本平均値よりも安定した代表値を提供します。
  2. 偏りの理解: 特に小規模なサンプルサイズでは、標本中央値が真の母中央値に対して偏りを持つ可能性があることを認識し、その影響を考慮した上で解釈を行う必要があります。
  3. SAS/Rによる実践: SASのPROC IMLやRのtidyverse (ggplot2, purrr)、rtablesを活用することで、標本中央値の特性をシミュレーションで確認し、また実データ解析で記述統計量として適切に利用できることを示しました。
  4. 国際的動向への対応: ICH E9(R1)などの最新の国際ガイダンスでは、平均値だけでなく中央値もEstimandとして認められており、日本の臨床統計担当者もこのような動向を理解し、解析戦略に反映させることが求められています。

臨床試験のデータ解析において、単に平均値のみに注目するのではなく、データの特性を深く理解し、標本中央値のようなロバストな統計量を適切に活用する視点は、より正確で信頼性の高い結論を導き出すために不可欠です。この記事が、皆様の臨床統計 の知識とスキル向上の一助となれば幸いです。
